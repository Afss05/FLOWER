# Documentation Site

Comprehensive documentation for FlowerShop eCommerce platform.

## Directory Structure

```
docs/
├── getting-started/          # Quick start guides
├── architecture/             # System design & patterns
├── api/                      # API documentation
├── frontend/                 # Frontend guides
├── backend/                  # Backend guides
├── database/                 # Database schema & design
├── deployment/               # Deployment guides
├── testing/                  # Testing strategies
├── multi-tenancy/            # Multi-tenant setup
├── configuration/            # Configuration guides
└── troubleshooting/          # Common issues & solutions
```

## Building Documentation

```bash
# Install dependencies
pnpm install

# Start docs server
pnpm dev

# Build docs
pnpm build
```

## Contributing Documentation

1. Create or update `.md` file in appropriate directory
2. Follow Markdown best practices
3. Include code examples where relevant
4. Keep content up-to-date with code changes
5. Cross-reference related topics

## Documentation Guidelines

- Use clear, concise language
- Include code examples
- Add diagrams for complex concepts
- Keep technical accuracy
- Update when features change

## Technology Stack

- **Docusaurus 3** for documentation site
- **Markdown** for content
- **Mermaid** for diagrams
- **MDX** for interactive examples
